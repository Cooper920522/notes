響應式系統

Vue 3 的響應式核心：ref、reactive、以及 Proxy 機制

ref — 基本類型響應式

import { ref } from 'vue'

// 建立響應式資料
const count = ref(0)
const name = ref('Vue')
const items = ref([])

// JS 中需要 .value 存取
count.value++
console.log(name.value)

// 模板中自動解包，不需要 .value
// <p>{{ count }}</p>  ← 直接使用
JS

reactive — 物件類型響應式

import { reactive } from 'vue'

const state = reactive({
  count: 0,
  user: { name: 'Alice', age: 25 }
})

// 直接存取，不需要 .value
state.count++
state.user.name = 'Bob'

// ⚠️ 不能解構 reactive！會失去響應性
// const { count } = state  ← 壞！count 不響應了

// ✅ 用 toRefs 安全解構
import { toRefs } from 'vue'
const { count, user } = toRefs(state)
JS
💡 ref vs reactive 怎麼選？ ref：基本類型（string, number, boolean）、或需要整體替換的物件
reactive：複雜物件，不需要整體替換
實務上很多人 統一用 ref，避免記憶規則

toRef、toRefs、isRef

import { toRef, toRefs, isRef, unref } from 'vue'

const state = reactive({ name: 'Vue', count: 0 })

// toRef — 單一屬性轉成 ref
const name = toRef(state, 'name')

// toRefs — 整個物件的屬性都轉成 ref
const { name, count } = toRefs(state)

// isRef — 判斷是否為 ref
isRef(count) // true

// unref — 取值 (是ref就.value，不是就直接回傳)
unref(count) // 等同 count.value
JS

shallowRef & shallowReactive（效能優化）

import { shallowRef, shallowReactive } from 'vue'

// 只有第一層是響應式，深層不追蹤
const bigList = shallowRef([...largeData])

// 觸發更新需要整體替換
bigList.value = [...bigList.value, newItem]
JS

Computed & Watch

計算屬性與偵聽器是響應式系統中最重要的工具

computed — 計算屬性

基於其他響應式資料衍生計算，有快取機制，依賴不變就不重算。

import { ref, computed } from 'vue'

const firstName = ref('John')
const lastName = ref('Doe')

// 只讀計算屬性
const fullName = computed(() => `${firstName.value} ${lastName.value}`)

// 可讀寫計算屬性
const fullNameRW = computed({
  get: () => `${firstName.value} ${lastName.value}`,
  set: (newVal) => {
    [firstName.value, lastName.value] = newVal.split(' ')
  }
})

// 使用
console.log(fullName.value) // 'John Doe'
fullNameRW.value = 'Jane Smith' // 觸發 set
JS
⚠️ 注意 computed 的 getter 不應有副作用（如發請求、改 DOM）。它只應計算並回傳值。

watch — 偵聽器

import { ref, watch, watchEffect } from 'vue'

const count = ref(0)

// 偵聽單個 ref
watch(count, (newVal, oldVal) => {
  console.log(`count: ${oldVal} → ${newVal}`)
})

// 偵聽多個來源
watch([firstName, lastName], ([newFirst, newLast]) => {
  console.log('名字變了')
})

// 偵聽物件內部（deep）
watch(() => state.user, (user) => {
  console.log(user)
}, { deep: true })

// 立即執行（immediate）
watch(count, handler, { immediate: true })

// 停止偵聽
const stop = watch(count, handler)
stop() // 手動停止
JS

watchEffect — 自動追蹤依賴

// 自動追蹤內部所有響應式依賴
const stop = watchEffect(() => {
  // 會追蹤 userId 和 token
  console.log(userId.value, token.value)
})

// 副作用清理（下次執行前呼叫）
watchEffect((onCleanup) => {
  const timer = setTimeout(() => fetchData(id.value), 300)
  onCleanup(() => clearTimeout(timer))
})
JS
特性computedwatchwatchEffect
用途衍生資料偵測變化執行副作用自動追蹤 + 副作用
快取✅ 有❌❌
回傳值計算結果無無
手動指定依賴自動✅ 手動自動
可取 oldVal❌✅❌

條件 & 列表渲染

控制 DOM 的顯示與生成

v-if / v-else-if / v-else

<div v-if="type === 'A'">類型 A</div>
<div v-else-if="type === 'B'">類型 B</div>
<div v-else>其他</div>

<!-- <template> 作為不渲染的包裝容器 -->
<template v-if="show">
  <h1>標題</h1>
  <p>內容</p>
</template>
HTML
💡 v-if vs v-show v-if：條件為 false 時完全不渲染 DOM（有更高切換代價）
v-show：永遠渲染，只切換 display:none（更適合頻繁切換）

v-for 列表渲染

<!-- 陣列 -->
<li v-for="(item, index) in items" :key="item.id">
  {{ index }}. {{ item.name }}
</li>

<!-- 物件 -->
<li v-for="(value, key, index) in obj" :key="key">
  {{ key }}: {{ value }}
</li>

<!-- 數字範圍 -->
<span v-for="n in 5" :key="n">{{ n }}</span>

<!-- v-for + v-if（v-if 優先，放在內部）-->
<template v-for="item in items" :key="item.id">
  <li v-if="item.active">{{ item.name }}</li>
</template>
HTML
⚠️ :key 很重要！ v-for 必須搭配唯一的 :key，幫助 Vue 辨識每個節點，避免渲染 bug。不要用 index 作為 key（在列表重排時會出問題）。

事件處理

基本事件綁定

Composition API 生命週期 Hooks

import {
  onBeforeMount, onMounted,
  onBeforeUpdate, onUpdated,
  onBeforeUnmount, onUnmounted,
  onErrorCaptured, onActivated, onDeactivated
} from 'vue'

onMounted(() => {
  // DOM 已掛載，可以操作 DOM、發 API 請求

onUnmounted(() => {
  // 清理：取消訂閱、清除 timer、移除 event listener

onErrorCaptured((err, instance, info) => {

})

Props & Emit

父子組件的溝通管道

defineProps

// 基本宣告
const props = defineProps(['title', 'count'])

// 帶型別驗證
const props = defineProps({
  title: String,
  count: {
    type: Number,
    required: true
  },
  items: {
    type: Array,
    default: () => []  // 物件/陣列必須用 factory function
  },
  status: {
    type: String,
    validator: (val) => ['active', 'inactive'].includes(val)
  }
})

// TypeScript 寫法（推薦）
const props = defineProps<{
  title: string
  count?: number
}>()

// 帶預設值的 TS（使用 withDefaults）
const props = withDefaults(defineProps<{ count?: number }>(), {
  count: 0
})
JS

defineEmits

// 基本宣告
const emit = defineEmits(['update', 'close'])

// 帶型別驗證
const emit = defineEmits({
  submit: (payload: { name: string }) => {
    return typeof payload.name === 'string'
  }
})

// 觸發事件
emit('update', newValue)
emit('close')

// 父組件監聽
<MyComp @update="handleUpdate" @close="handleClose" />

Provide & Inject

跨越組件層級的依賴注入，解決 Props Drilling 問題

// 祖先組件 provide
import { provide, ref } from 'vue'

const theme = ref('dark')
provide('theme', theme)  // 提供響應式值
provide('setTheme', (v) => theme.value = v)  // 提供修改函數

// 任意深度子孫組件 inject
import { inject } from 'vue'

const theme = inject('theme')         // 接收
const theme = inject('theme', 'light') // 帶預設值
const setTheme = inject('setTheme')

// 使用 Symbol 作為 key（避免命名衝突，推薦）
// injection-keys.js
export const THEME_KEY = Symbol('theme')
JS
💡 建議保持資料流向清晰 修改 provide 的資料應由提供方負責。在子組件中直接修改 inject 進來的響應式資料雖然可行，但會破壞單向資料流，難以追蹤。建議 provide 一個修改函數給子組件使用。

Composables 組合式函數

Vue 3 最重要的邏輯複用模式，取代 Vue 2 的 Mixins

什麼是 Composable？

Composable 是一個函數，封裝並複用有狀態的邏輯。它利用 Composition API 的響應式系統，提供類似 React Hooks 的體驗。

實作範例 — useFetch

// composables/useFetch.js
import { ref, watchEffect } from 'vue'

export function useFetch(url) {
  const data = ref(null)
  const error = ref(null)
  const loading = ref(false)

  watchEffect(async () => {
    data.value = null
    error.value = null
    loading.value = true
    try {
      data.value = await (await fetch(toValue(url))).json()
    } catch (e) {
      error.value = e
    } finally {
      loading.value = false
    }
  })

  return { data, error, loading }
}

// 組件中使用
const { data, loading, error } = useFetch('/api/users')

實作範例 — useLocalStorage

export function useLocalStorage(key, defaultValue) {
  const stored = localStorage.getItem(key)
  const value = ref(stored ? JSON.parse(stored) : defaultValue)

  watch(value, (newVal) => {
    localStorage.setItem(key, JSON.stringify(newVal))
  }, { deep: true })

  return value
}

// 使用
const theme = useLocalStorage('theme', 'dark')
JS

Composable 命名慣例

  • 函數名稱以 use 開頭（如 useCounter、useAuth）
  • 放在 composables/ 資料夾
  • 只能在 <script setup> 或 setup() 頂層呼叫
  • 不能在條件判斷或迴圈中呼叫

自訂指令

當你需要直接操作 DOM 時，自訂指令是最佳選擇

// 自訂指令的 hooks 與生命週期一致
const vFocus = {
  mounted(el) { el.focus() }
}

// 在 script setup 中，v 開頭的變數即為自訂指令
// <input v-focus />

// 完整 hooks
const vColor = {
  created(el, binding, vnode) {},
  beforeMount(el, binding) {},
  mounted(el, binding) {
    // binding.value — 指令的值（v-color="'red'"）
    // binding.arg   — 修飾符前的參數（v-color:bg）
    // binding.modifiers — 修飾符物件（v-color.bold）
    el.style.color = binding.value
  },
  beforeUpdate(el, binding) {},
  updated(el, binding) {},
  beforeUnmount(el) {},
  unmounted(el) {}
}

// 全域註冊
app.directive('highlight', {
  mounted(el, binding) {
    el.style.backgroundColor = binding.value || 'yellow'
  }
})
// 使用：<p v-highlight="'pink'">...</p>

基本設定

// router/index.js
import { createRouter, createWebHistory } from 'vue-router'

const router = createRouter({
  history: createWebHistory(),  // HTML5 History 模式
  routes: [
    { path: '/', component: HomeView },
    { path: '/users/:id', component: UserView },
    {
      path: '/admin',
      component: AdminLayout,
      meta: { requiresAuth: true },
      children: [
        { path: '', component: AdminHome },
        { path: 'users', component: AdminUsers }
      ]
    },
    { path: '/:pathMatch(.*)*', component: NotFound }
  ]

Pinia 狀態管理

Vue 3 官方推薦的狀態管理庫，取代 Vuex

定義 Store

// stores/counter.js
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

// Setup Store 風格（推薦，類似 script setup）
export const useCounterStore = defineStore('counter', () => {
  // state
  const count = ref(0)

  // getters
  const doubled = computed(() => count.value * 2)

  // actions
  function increment() { count.value++ }
  async function fetchCount() {
    count.value = await api.getCount()
  }

  return { count, doubled, increment, fetchCount }
})
JS

使用 Store

<script setup>
import { useCounterStore } from '@/stores/counter'
import { storeToRefs } from 'pinia'

const store = useCounterStore()

// ✅ 用 storeToRefs 解構響應式資料（保持響應性）
const { count, doubled } = storeToRefs(store)

// actions 直接解構即可（函數不需要響應性）
const { increment } = store
</script>

<template>
  <p>{{ count }} (x2 = {{ doubled }})</p>
  <button @click="increment">+1</button>
</template>

Nuxt 3 整合

基於 Vue 3 的全端框架，支援 SSR / SSG / SPA

Nuxt 3 目錄結構

├── pages/          ← 自動路由（約定式路由）
│   ├── index.vue   ← /
│   ├── about.vue   ← /about
│   └── users/
│       └── [id].vue ← /users/:id（動態路由）
├── components/     ← 自動 import，無需手動 import
├── composables/    ← 自動 import
├── layouts/        ← 佈局組件
├── server/         ← 伺服器端 API Routes
│   └── api/
│       └── users.get.ts
├── plugins/        ← 插件（在應用啟動時執行）
├── middleware/     ← 路由中間件
├── stores/         ← Pinia stores（自動 import）
└── nuxt.config.ts
