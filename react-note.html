React 完整學習筆記

從零開始，系統化掌握 React 18 全部核心概念，以函數組件 + Hooks 為主軸

什麼是 React？

React 是由 Meta（Facebook）開發的 JavaScript 函式庫，專注於建構使用者介面。它不是完整的框架，而是 UI 層的解決方案，透過組件化與宣告式的方式描述 UI。

🧩

組件化

UI 拆分成獨立、可複用的組件，每個組件管理自己的狀態

📜

宣告式

描述 UI 應該長什麼樣，React 負責更新 DOM

⚡

Virtual DOM

高效的 diff 演算法，最小化真實 DOM 操作

🔄

單向資料流

資料從父到子流動，狀態變化可預測

React 18 新特性

特性說明
Concurrent Mode並發渲染，讓 React 能中斷、暫停、恢復渲染
Automatic Batching自動批次更新 state，減少不必要的重渲染
Transitions區分緊急與非緊急更新，useTransition / startTransition

純組件原則

React 組件應該是純函數：相同的 props 輸入永遠產生相同的 JSX 輸出，不應有副作用。

✅ 純組件

  return (
    <ul>
      {items.map(i => (
        <li key={i.id}>
          {i.name}
        </li>
      ))}
    </ul>
  )
}

❌ 有副作用

let count = 0

function Impure() {
  count++ // 修改外部變數！
  return <p>{count}</p>

State & useState

組件的內部狀態，state 改變觸發重新渲染

useState 基礎

import { useState } from 'react'

function Counter() {
  // [當前值, 更新函數] = useState(初始值)
  const [count, setCount] = useState(0)
  const [name, setName] = useState('')
  const [user, setUser] = useState({ name: 'Alice', age: 25 })

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  )
}
JSX

State 更新規則

// ✅ 使用 updater function（依賴前一個 state 時必用）
setCount(prev => prev + 1)

// ❌ 危險！在批次更新中可能拿到舊值
setCount(count + 1)
setCount(count + 1) // 這兩行實際上只加一次

// ✅ updater function 正確連續更新
setCount(prev => prev + 1)
setCount(prev => prev + 1) // 正確加兩次

// 物件 state — 必須展開，不能直接修改
setUser(prev => ({ ...prev, age: 26 }))  // ✅
user.age = 26 // ❌ 直接修改，不會觸發重渲染

// 陣列 state — 使用不可變操作
setItems(prev => [...prev, newItem])          // 新增
setItems(prev => prev.filter(i => i.id !== id)) // 刪除
setItems(prev => prev.map(i =>               // 更新
  i.id === id ? { ...i, done: true } : i
))
JSX

Lazy Initial State

// 計算成本高的初始值，傳函數而非值
// 只有第一次渲染才會執行，之後不再呼叫
const [state, setState] = useState(() => {
  return expensiveCalculation()
})
JS
💡 React vs Vue 的思維差異 Vue 的 ref 直接 .value = 新值就能觸發更新。React 的 state 必須透過 setter 函數，且永遠不能直接修改 state 物件（Immutability 原則）。

useEffect

處理副作用：API 請求、訂閱、手動 DOM 操作、計時器

基本用法

import { useEffect } from 'react'

function App() {
  // 每次渲染後都執行
  useEffect(() => {
    document.title = 'Hello'
  })

  // 只在掛載時執行一次（[] = 沒有依賴）
  useEffect(() => {
    fetchData()
  }, [])

  // userId 改變時執行
  useEffect(() => {
    fetchUser(userId)
  }, [userId])
}
JS

Cleanup 清理函數

useEffect(() => {
  // 設定訂閱
  const subscription = subscribe(userId)

  // 回傳清理函數（下次 effect 執行前 or 卸載時呼叫）
  return () => {
    subscription.unsubscribe()
  }
}, [userId])

// timer 範例
useEffect(() => {
  const id = setInterval(() => {
    setCount(c => c + 1)
  }, 1000)
  return () => clearInterval(id)
}, [])
JS

常見使用場景對照

情境依賴陣列說明
掛載時執行[]等同 onMounted
每次渲染無第二參數謹慎使用
依賴變化[dep1, dep2]依賴更新時執行
卸載清理[] + return fn等同 onUnmounted
⚠️ 常見錯誤：無限迴圈 如果在 useEffect 內部更新了某個 state，而這個 state 又在依賴陣列中，就會造成無限迴圈。解決方式：用 updater function、或重新思考邏輯結構。

useEffect 與 API 請求

useEffect(() => {
  let cancelled = false

  async function fetchData() {
    try {
      setLoading(true)
      const res = await fetch(`/api/users/${userId}`)
      const data = await res.json()
      if (!cancelled) setUser(data)  // 防止 race condition
    } catch (err) {
      if (!cancelled) setError(err)
    } finally {
      if (!cancelled) setLoading(false)
    }
  }

  fetchData()
  return () => { cancelled = true }  // 清理：防止 stale closure
}, [userId])

useContext

跨層級傳遞資料，解決 Props Drilling 問題

import { createContext, useContext, useState } from 'react'

// 1. 建立 Context（通常在獨立檔案）
const ThemeContext = createContext('light')  // 預設值

// 2. Provider 包裝需要使用的組件
function App() {
  const [theme, setTheme] = useState('dark')
  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      <Layout />
    </ThemeContext.Provider>
  )
}

// 3. 任意深度的子組件使用
function Button() {
  const { theme, setTheme } = useContext(ThemeContext)
  return (
    <button onClick={() => setTheme(t => t === 'dark' ? 'light' : 'dark')}>
       目前：{theme}
    </button>
  )
}
JSX

封裝成自訂 Hook（最佳實踐）

// context/ThemeContext.jsx
const ThemeContext = createContext(null)

export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light')
  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  )
}

// 封裝 hook，並加入保護
export function useTheme() {
  const ctx = useContext(ThemeContext)
  if (!ctx) throw new Error('useTheme 必須在 ThemeProvider 內使用')
  return ctx
}
JSX
💡 Context 的效能考量 Context value 改變時，所有使用 useContext 的子組件都會重新渲染。如果 value 是物件，記得用 useMemo 包裝，或拆分成多個 Context（讀取與寫入分開）。

useReducer

複雜 state 邏輯的最佳選擇，類似 Redux 的模式

import { useReducer } from 'react'

// 1. 定義初始 state
const initialState = { count: 0, step: 1 }

// 2. 定義 reducer（純函數）
function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { ...state, count: state.count + state.step }
    case 'decrement':
      return { ...state, count: state.count - state.step }
    case 'setStep':
      return { ...state, step: action.payload }
    case 'reset':
      return initialState
    default:
      throw new Error(`未知 action: ${action.type}`)
  }
}

// 3. 組件中使用
function Counter() {
  const [state, dispatch] = useReducer(reducer, initialState)

  return (
    <div>
      <p>計數：{state.count}，步進：{state.step}</p>
      <button onClick={() => dispatch({ type: 'increment' })}>+</button>
      <button onClick={() => dispatch({ type: 'setStep', payload: 5 })}>步進設為5</button>
    </div>
  )
}
JSX
特性useStateuseReducer
適用場景簡單獨立的 state複雜、相關聯的 state
更新方式直接 setterdispatch action
測試較難單元測試reducer 純函數，易測試
可讀性簡單邏輯集中，更清晰

useMemo & useCallback

記憶化計算結果與函數，避免不必要的重新計算

useMemo — 記憶化計算值

import { useMemo } from 'react'

function ProductList({ products, filterText }) {
  // 只有 products 或 filterText 改變時才重新過濾
  const filteredProducts = useMemo(() => {
    return products.filter(p =>
      p.name.includes(filterText)
    )
  }, [products, filterText])

  return <ul>{filteredProducts.map(p => <li key={p.id}>{p.name}</li>)}</ul>
}
JSX

useCallback — 記憶化函數

import { useCallback } from 'react'

function Parent() {
  const [count, setCount] = useState(0)

  // 沒有 useCallback：每次 Parent 重渲染，handleClick 都是新函數
  // Child 因為 props 改變而跟著重渲染

  // 有 useCallback：count 不變時，handleClick 是同一個函數引用
  const handleClick = useCallback(() => {
    console.log('clicked', count)
  }, [count])

  return <Child onClick={handleClick} />
}

// Child 用 React.memo 包裝才有意義
const Child = React.memo(({ onClick }) => {
  console.log('Child render')
  return <button onClick={onClick}>點我</button>
})
JSX
⚠️ 不要過度使用 useMemo 和 useCallback 本身有計算成本。只在確認有效能問題時使用，不要預防性地到處加。大多數情況下，React 的重渲染已經足夠快。

自訂 Hooks

提取並複用有狀態的邏輯，React 最強大的邏輯複用方式

useWindowSize

function useWindowSize() {
  const [size, setSize] = useState({
    width: window.innerWidth,
    height: window.innerHeight
  })

  useEffect(() => {
    function handleResize() {
      setSize({ width: window.innerWidth, height: window.innerHeight })
    }
    window.addEventListener('resize', handleResize)
    return () => window.removeEventListener('resize', handleResize)
  }, [])

  return size
}

// 使用
const { width, height } = useWindowSize()
JS

useDebounce

function useDebounce(value, delay = 500) {
  const [debouncedValue, setDebouncedValue] = useState(value)

  useEffect(() => {
    const timer = setTimeout(() => {
      setDebouncedValue(value)
    }, delay)
    return () => clearTimeout(timer)
  }, [value, delay])

  return debouncedValue
}

// 使用 — 搜尋框防抖
const [query, setQuery] = useState('')
const debouncedQuery = useDebounce(query, 300)

useEffect(() => {
  searchAPI(debouncedQuery)  // 只有停止輸入 300ms 後才發請求
}, [debouncedQuery])
JS

自訂 Hook 規則

  • 名稱必須以 use 開頭
  • 只能在函數組件或其他 Hook 的頂層呼叫
  • 不能在條件式或迴圈中呼叫
  • 可以呼叫其他 Hooks

表單處理

受控組件 vs 非受控組件

受控組件（Controlled）

// state 是唯一資料來源
function LoginForm() {
  const [form, setForm] = useState({ email: '', password: '' })

  function handleChange(e) {
    setForm(prev => ({ ...prev, [e.target.name]: e.target.value }))
  }

  function handleSubmit(e) {
    e.preventDefault()
    console.log(form)
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        name="email"
        value={form.email}
        onChange={handleChange}
      />
      <input
        name="password"
        type="password"
        value={form.password}
        onChange={handleChange}
      />
      <button type="submit">登入</button>
    </form>
  )
}

  function handleSearch(e) {
    setQuery(e.target.value)  // 緊急：立即更新輸入框

    startTransition(() => {
      setResults(expensiveSearch(e.target.value))  // 非緊急：可被中斷
    })
  }

  return (
    <div>
      <input value={query} onChange={handleSearch} />
      {isPending ? <Spinner /> : <ResultList results={results} />}
    </div>
  )
}

// 查詢資料
function UserList() {
  const { data, isLoading, error, refetch } = useQuery({
    queryKey: ['users'],
    queryFn: () => fetch('/api/users').then(r => r.json()),
    staleTime: 5 * 60 * 1000,  // 5分鐘內不重新獲取
  })

  if (isLoading) return <Spinner />
  if (error) return <Error message={error.message} />
  return <ul>{data.map(u => <li key={u.id}>{u.name}</li>)}</ul>
}

// 修改資料（POST/PUT/DELETE）
function AddUser() {
  const mutation = useMutation({
    mutationFn: (newUser) => fetch('/api/users', {
      method: 'POST', body: JSON.stringify(newUser)
    }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['users'] })
    }
  })

  return <button onClick={() => mutation.mutate({ name: 'Alice' })}>新增</button>
}
